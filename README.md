# Ejecuciones de codigo 

Fase 1 - Primeramente se debe ejecutar esta fase, ya que es una de las mas importantes, donde se cargan los datos para analizar y preprocesar el dataset de MNIST, leyendo los datos que contiene el mismo modelo y tambien preprocesar los datos, para que el codigo sea normalizado, y utilizado para el entrenamiento de los distintos modelos a ocupar dentro del proyecto, tambien, si no se entrena en la primera fase, el modelo no aprendera bien

Fase 2 - Como segunda ejecucion, se debe ejecutar esta fase, para entrenar y probar modelos no optimizados, tambien realizando experimentos controlados, como cambio de learning rate, cambio de activaciones y salida y por ultimo, cambios en las epocas dentro de los experimentos, probando distintos batch-size, learning rates y epocas.

Fase 3 - Para esta fase 3, se evaluan los modelos recientemente entrenados, para probar distintas metricas dentro del modelo a mejorar, tambien se prueban distintas ejecuciones de codigo

Fase 4 - Para la fase final, se debe ejecutar la fase de el codigo con el modelo optimizado, ya que esta fase, corresponde a la final, donde tambien se evaluan distintas metricas comparadas con ambos modelos, finalmente aplicando una tabla comparativa de todas las metricas de todos los modelos utilizados a lo largo del proyecto
